## 如何使用

### 方法一：直接联机（推荐先尝试）

1. **房主创建房间**
   - 点击「🏠 创建房间」
   - 选择关卡
   - 复制房间码给好友

2. **好友加入**
   - 点击「🔗 加入房间」
   - 输入房间码
   - 点击「🔍 检测网络环境」查看支持情况
   - 点击「加入」

3. **如果连接失败**
   - 双方都点击「🔍 检测网络环境」
   - 查看是否显示「✓ 支持跨WiFi」
   - 如果不支持，使用方法二或三

---

### 方法二：手机热点（最稳定）

1. 房主打开手机热点
2. 所有玩家连接该热点
3. 按正常流程创建/加入房间
4. 此时等同于同一WiFi，100%连通

---

### 方法三：使用第三方工具（100%成功）

#### 方案A：Radmin VPN（免费）
1. 所有玩家下载 [Radmin VPN](https://www.radmin-vpn.com/)
2. 创建一个网络/加入同一网络
3. 此时等同于同一局域网
4. 正常使用房间码联机

#### 方案B：Parsec（免费）
1. 下载 [Parsec](https://parsec.app/)
2. 房主分享链接
3. 好友通过链接加入
4. 适合屏幕共享+语音

#### 方案C：ZeroTier（免费）
1. 注册 [ZeroTier](https://www.zerotier.com/)
2. 创建虚拟网络
3. 所有玩家加入该网络
4. 获得虚拟IP，直接联机

## 技术说明

### NAT类型与连通性

| NAT类型 | STUN支持 | TURN支持 | 跨WiFi连通 |
|---------|----------|----------|------------|
| 开放型 | ✓ | ✓ | ✓ |
| 完全锥型 | ✓ | ✓ | ✓ |
| 受限锥型 | △ | ✓ | ✓ |
| 端口受限 | △ | ✓ | ✓ |
| 对称型 | ✗ | ✓ | ✓ |

### 当前配置
- **STUN服务器**: Google STUN (3个)
- **TURN服务器**: Open Relay (3个)
- **信令服务器**: PeerJS公共服务器

### 如果TURN也不通
说明双方都是对称型NAT且TURN服务器满载，此时：
1. 使用手机热点
2. 或部署私有TURN服务器
3. 或使用第三方VPN工具

## 常见问题

**Q: 为什么同一WiFi能连，不同WiFi不行？**
A: 局域网内直接通信，跨网络需要NAT穿透。

**Q: TURN服务器免费吗？**
A: 使用的是Open Relay免费服务，可能有速率限制。

**Q: 会泄露隐私吗？**
A: 游戏数据点对点传输，不经过服务器。TURN仅中继加密数据。

**Q: 延迟高吗？**
A: 同一WiFi < 10ms，跨WiFi直连 < 50ms，TURN中继 < 150ms。

## 部署私有TURN服务器（高级）

如果需要更稳定的跨WiFi体验，可以部署自己的TURN服务器：

```bash
# 使用coturn（需要VPS）
docker run -d --network=host \
  -e MIN_PORT=10000 \
  -e MAX_PORT=20000 \
  -e USERNAME=youruser \
  -e PASSWORD=yourpass \
  coturn/coturn
```

然后在代码中添加：
```javascript
{
  urls: 'turn:你的服务器IP:3478',
  username: 'youruser',
  credential: 'yourpass'
}
```
